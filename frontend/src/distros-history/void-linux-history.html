Void Linux History

Void Linux is an independent Linux distribution that uses the X Binary Package System (XBPS) package manager, which was designed and implemented from scratch, and the runit init system. The distribution is available for the Intel x86, ARM, and MIPS architectures with a focus on simplicity, clarity, and minimalism.

  • 2008 – The Beginning

    Void Linux was created in 2008 by Juan Romero Pardines, a former developer of NetBSD. The initial release focused on providing a simple and minimalistic Linux distribution.

  • 2012 – XBPS Package Manager

    In 2012, Void Linux introduced the X Binary Package System (XBPS), a new package manager designed and implemented from scratch to provide a fast and efficient way to manage software packages.

  • 2014 – Runit Init System

    In 2014, Void Linux adopted the runit init system, replacing the traditional SysV init system. Runit is known for its simplicity and speed, aligning with Void Linux's philosophy.

  • 2016 – ARM and MIPS Support

    In 2016, Void Linux expanded its support to include ARM and MIPS architectures, making it available for a wider range of devices and users.

  • 2018 – Musl Support

    In 2018, Void Linux introduced support for the musl C library, providing an alternative to the GNU C Library (glibc) and further enhancing the distribution's flexibility and performance.

  • 2020 – Continued Development

    Void Linux continues to be actively developed and maintained by a dedicated community of developers and users, focusing on simplicity, clarity, and minimalism.
